So wird getestet, ob ein Knoten zugänglich ist

Um GlassFish Server-Instanzen auf einem Knoten zu starten, der einen Remote-Host darstellt, muss der DAS Verbindung mit dem Knoten aufnehmen können. Bevor Sie versuchen, Instanzen auf einem derartigen Knoten zu erstellen oder zu starten, testen Sie, ob der DAS Verbindung mit dem Knoten aufnehmen kann.

Der Knotentyp, den Sie testen, muss DCOM oder SSH sein. Sie können keinen Knoten vom Typ CONFIG testen, weil ein Knoten vom Typ CONFIG nicht für die Remote-Kommunikation aktiviert ist.

Bevor Sie beginnen

Stellen Sie sicher, dass das entsprechende Protokoll für den Knotentyp auf dem Host konfiguriert ist, den der Knoten darstellt. Der Knoten ist für die Remote-Kommunikation über dieses Protokoll aktiviert.

  • Wenn der Knotentyp DCOM ist, muss das Distributed Component Object Model-(DCOM-)Protokoll konfiguriert sein.

  • Wenn der Knotentyp SSH ist, muss das Secure Shell-(SSH-)Protokoll konfiguriert sein.

    1. Wählen Sie in der Navigationsbaumstruktur den Knoten "Knoten".

      Die "Knoten"-Seite wird geöffnet. Auf dieser Seite werden alle Knoten in der aktuellen Domain aufgelistet.

    2. Folgen Sie in der Knotentabelle dem Ping-Link für den Knoten, den Sie testen.

      Eine Meldung, ob der DAS Verbindung mit dem Knoten aufnehmen konnte, wird der Seite "Knoten" hinzugefügt.
