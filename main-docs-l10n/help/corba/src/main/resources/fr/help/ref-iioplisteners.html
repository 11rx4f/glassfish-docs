Processus d'écoute IIOP

La page Processus d'écoute IIOP permet de configurer les processus d'écoute IIOP.

Certains processus d'écoute IIOP peuvent être configurés pour un ORB, chacun acceptant des connexions sur différentes combinaisons d'adresses réseau et de ports. Les processus d'écoute peuvent être configurés différemment pour des communications sécurisées et l'authentification client.

Par défaut, trois processus d'écoute IIOP sont configurés : orb-listener-1, SSL et SSL_MUTUALAUTH.

Pour chaque processus d'écoute IIOP, la page Processus d'écoute IIOP contient les informations ci-après.

Nom

Nom du processus d'écoute IIOP.

Activé

True si le processus d'écoute IIOP est activé ou False si le processus d'écoute est désactivé.

Adresse réseau

Adresse IP (Internet Protocol) ou nom d'hôte DNS pouvant être résolu par le processus d'écoute IIOP.

Port du processus d'écoute

Numéro de port du processus d'écoute IIOP.

  • Modification des paramètres SSL pour un processus d'écoute IIOP sécurisé
