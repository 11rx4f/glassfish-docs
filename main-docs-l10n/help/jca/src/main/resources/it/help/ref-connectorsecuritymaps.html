Mappe di sicurezza connection pool connettore

Utilizzare la Mappe di sicurezza connection pool connettore per gestire le mappe di sicurezza per i connection pool del connettore.

Le mappe di sicurezza per i connection pool del connettore consentono la creazione di un mapping tra utenti e gruppi Jakarta EE e utenti e gruppi EIS. Una mappa di sicurezza consente di eseguire il mapping tra il principal ricevuto durante l'autenticazione di servlet o di un EJB e le credenziali accettate dal sistema EIS. È possibile mappare più principal di GlassFish Server allo stesso principal backend.

La mappa di sicurezza per un connection pool è diversa dalla mappa di sicurezza di lavoro del connettore, che esegue il mapping tra un principal associato a un'istanza di lavoro in entrata e un principal in un realm di sicurezza del dominio di GlassFish Server. Per informazioni sulle mappe di sicurezza lavoro del connettore, vedere Mappe di sicurezza lavoro.

Per ogni mappa di sicurezza vengono fornite le informazioni riportate di seguito.

Nome

Il nome della mappa di sicurezza.

La tabella Mappe di sicurezza contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare una nuova mappa di sicurezza.

Elimina

Pulsante che consente di eliminare una o più mappe di sicurezza selezionate.
