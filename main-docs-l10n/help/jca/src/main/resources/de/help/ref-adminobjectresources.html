Admin-Objektressourcen

Auf der Seite "Admin-Objektressourcen" können Sie verwaltete Objektressourcen konfigurieren.

Eine verwaltete Objektressource bietet spezielle Funktionen, die vom Ressourcenadapter für das bereitgestellte Connector-Modul definiert werden.

Für jede verwaltete Objektressource werden die folgenden Informationen zur Verfügung gestellt.

JNDI-Name

Ein eindeutiger Name, der die verwaltete Objektressource identifiziert.

Aktiviert

(Diese Spalte wird angezeigt, wenn nur die Standardserverinstanz server in der Domain vorhanden ist.)

Zeigt ein Häkchen an, wenn die Ressource auf dem Ziel aktiviert ist, oder ein X, wenn die Ressource deaktiviert ist.

Status

(Diese Spalte wird angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.)

Die Anzahl Cluster und Standalone-Instanzen, auf die die Ressource ausgerichtet wird, und auf wie vielen dieser Ziele die Ressource aktiviert ist. Beispiel: "Auf 2 von 4 Ziel(en) aktiviert" bedeutet, dass die Ressource auf vier Cluster und Standalone-Instanzen ausgerichtet und auf zweien dieser vier Ziele aktiviert ist.

Ressourcentyp

Der Java-Typ der verwalteten Objektressource.

Beschreibung

Eine Beschreibung der verwalteten Objektressource.

Die Tabelle "Ressourcen" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen einer neuen verwalteten Objektressource.

Löschen

Schaltfläche zum Löschen von mindestens einer ausgewählten verwalteten Objektressource.

Aktivieren

Schaltfläche zum Aktivieren von mindestens einer ausgewählten verwalteten Objektressource.

Deaktivieren

Schaltfläche zum Deaktivieren von mindestens einer ausgewählten verwalteten Objektressource.

Verwandte Aufgaben

  • So wird eine verwaltete Objektressource erstellt

  • So wird eine verwaltete Objektressource bearbeitet

  • So wird eine verwaltete Objektressource gelöscht
