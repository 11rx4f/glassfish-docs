Módulos de Ciclo de Vida

Utilice la página Módulos de Ciclo de Vida para configurar los módulos de ciclo de vida.

Un módulo de ciclo de vida realiza tareas cuando lo dispara uno o más eventos del ciclo de vida del servidor. Los posibles eventos del servidor que pueden dispararlo son: inicialización, inicio, listo para atender solicitudes y apagado. Los módulos de ciclo de vida no forman parte de la especificación de Jakarta EE, sino que son una mejora de GlassFish Server.

Se proporciona la siguiente información para cada módulo de ciclo de vida.

Nombre

Nombre del módulo de ciclo de vida.

Activado o Estado

Una marca de control si la aplicación está activada o una X si está desactivada, en el caso de que sólo exista la instancia de servidor por defecto, server.

Si existen clusters u otras instancias de servidores independientes, seleccione Activado en Todos los Destinos para ver los destinos en los que está desplegada la aplicación.

Orden de Carga

Valor que especifica el orden relativo en el que el módulo de ciclo de vida se debe cargar.

La tabla Módulos de Ciclo de Vida también contiene las siguientes opciones.

Nuevo

Botón para crear un nuevo módulo de ciclo de vida.

Suprimir

Botón para suprimir uno o más módulos de ciclo de vida seleccionados.

Activar

Botón para activar uno o más módulos de ciclo de vida seleccionados. Sólo está presente si existe la instancia de servidor por defecto, server.

Desactivar

Botón para desactivar uno o más módulos de ciclo de vida seleccionados. Sólo está presente si existe la instancia de servidor por defecto, server.
