Utilizar el módulo de auditoría por defecto

El módulo de auditoría default registra solicitudes de autenticación y de autorización en el archivo log del servidor. Para obtener información sobre cómo cambiar la ubicación del archivo log, consulte Configurar valores de registro generales.

Las entradas de log de autenticación incluyen la información siguiente:

  • Nombres de usuarios que han intentado autenticarse

  • El dominio que ha procesado la solicitud de acceso

  • El URI del módulo web o componente EJB solicitado

  • Éxito o fallo de la solicitud

Independientemente de si se activa el registro de auditoría, GlassFish Server registra todos los eventos de autenticación denegados.

Las entradas de log de autorización incluyen la información siguiente:

  • Los nombres de los usuarios autenticados, si hay alguno

  • El URI web o componente EJB solicitado

  • Solicitudes correctas o con fallos

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la instancia que va a configurar:

    • Para configurar una instancia determinada, amplíe el nodo de configuración de la instancia. Por ejemplo, para la instancia por defecto server, amplíe el nodo server-config.

    • Para configurar los valores por defecto para todas las instancias, amplíe el nodo default-config.

  3. En el nodo Configuración, seleccione el nodo Seguridad.

    Se abrirá la página Seguridad.

  4. En la lista desplegable Módulos de Auditoría, seleccione el módulo de auditoría llamado default.

  5. Haga clic en Guardar.
