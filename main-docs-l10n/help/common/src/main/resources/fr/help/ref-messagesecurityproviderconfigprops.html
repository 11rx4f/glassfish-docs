Propriétés propres aux configurations de fournisseur de sécurité des messages

Les propriétés suivantes sont disponibles pour la configuration d'un fournisseur de sécurité des messages.

server.config

Répertoire et nom d'un fichier XML qui contient les informations de configuration du serveur. La valeur par défaut est domain-dir/config/wss-server-config-1.0.xml.

debug

La valeur true active le vidage des messages de débogage de fournisseur côté serveur vers le journal du serveur. La valeur par défaut est false.

dynamic.username.password

La valeur true signale à l'exécution du fournisseur qu'elle doit collecter le nom utilisateur et le mot de passe de CallbackHandler pour chaque demande. Si la valeur est False, le nom utilisateur et le mot de passe de wsse:UsernameToken(s) sont collectés une seule fois, lors de l'initialisation du module. Cette propriété s'applique uniquement à ClientAuthModule. La valeur par défaut est false.

encryption.key.alias

Spécifie la clé de cryptage utilisée par le fournisseur. La clé est identifiée par son alias de keystore. La valeur par défaut est s1as.

signature.key.alias

Spécifie la clé de signature utilisée par le fournisseur. La clé est identifiée par son alias de keystore. La valeur par défaut est s1as.
