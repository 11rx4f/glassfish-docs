Modules d'audit

Utilisez la page Modules d'audit pour configurer les modules d'audit.

L'audit est la manière dont GlassFish Server consigne les événements importants, comme les erreurs et les brèches de sécurité. GlassFish Server consigne tous les événements d'authentification dans des fichiers journaux. Un journal d'accès complet fournit un suivi chronologique des événements d'accès dans GlassFish Server.

Pour chaque module d'audit, les informations ci-après sont fournies.

Nom

Nom du module d'audit.

Nom de classe

Nom qualifié complet de la classe qui implémente ce module. Le nom de classe du module d'audit par défaut est com.sun.enterprise.security.Audit.

La table Modules contient également les options ci-après.

Nouveau

Bouton permettant de créer un module d'audit.

Supprimer

Bouton permettant de supprimer les modules d'audit sélectionnés.

Commandes asadmin connexes

  • create-audit-module(1)

  • list-audit-modules(1)

  • delete-audit-module(1)
