Auditmodule

Auf der Seite "Auditmodule" können Sie Auditmodule konfigurieren.

Beim Auditing wird überwacht, wie GlassFish Server wichtige Ereignisse, z.B. Fehler oder Sicherheitsverletzungen, aufzeichnet. GlassFish Server zeichnet sämtliche Authentifizierungsereignisse in Logdateien auf. Ein vollständiges Zugriffslog umfasst eine Reihe von GlassFish Server-Zugriffsereignissen.

Folgende Informationen werden für jedes Auditmodul zur Verfügung gestellt.

Name

Der Name des Auditmoduls.

Klassenname

Der vollständig angegebene Name der Klasse, die dieses Modul implementiert. Der Klassenname für das Standardauditmodul lautet com.sun.enterprise.security.Audit.

Die Tabelle "Module" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines neuen Auditmoduls.

Löschen

Schaltfläche zum Löschen von mindestens einem Auditmodul.

Zugehörige asadmin-Befehle
