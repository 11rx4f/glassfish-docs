Nomes Lógicos JNDI

Quando você cria um recurso, especifica um nome JNDI exclusivo para ele. As aplicações usam esse nome para acessar o recurso.

O padrão Jakarta EE especifica que determinados recursos default sejam disponibilizados para aplicações, e define nomes JNDI específicos para esses recursos default. O GlassFish Server disponibiliza esses nomes por meio do uso de nomes JNDI lógicos, que mapeiam os nomes JNDI do padrão Jakarta EE para recursos específicos do GlassFish Server, da seguinte forma:

java:comp/DefaultContextService

Esse nome de padrão Jakarta EE é mapeado para o recurso concurrent/__defaultContextService.

java:comp/DefaultManagedThreadFactory

Esse nome de padrão Jakarta EE é mapeado para o recurso concurrent/__defaultManagedThreadFactory.

java:comp/DefaultManagedExecutorService

Esse nome de padrão Jakarta EE é mapeado para o recurso concurrent/__defaultManagedExecutorService.

java:comp/DefaultManagedScheduledExecutorService

Esse nome de padrão Jakarta EE é mapeado para o recurso concurrent/__defaultManagedScheduledExecutorService.

java:comp/DefaultDataSource

Esse nome de padrão Jakarta EE é mapeado para o recurso jdbc/__default.

java:comp/DefaultJMSConnectionFactory

Esse nome de padrão Jakarta EE é mapeado para o recurso jms/__defaultConnectionFactory.
