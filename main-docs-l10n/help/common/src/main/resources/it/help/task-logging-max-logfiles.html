Per modificare il limite al numero dei file di log sottoposti a rotazione

Quando esegue la rotazione di un file di log, GlassFish Server crea un nuovo file vuoto denominato server.log e rinomina il file precedente server.log_data, dove data corrisponde alla data e all'ora di rotazione del file.

Per impostazione predefinita il numero di file di log di &ProductName sottoposti a rotazione è limitato a 10. Quando viene raggiunto tale limite, il file di log meno recente viene eliminato quando GlassFish Server esegue la rotazione successiva di un file di log.

Se necessario è possibile modificare il numero limite dei file di log sottoposti a rotazione. Il nuovo limite impostato verrà applicato ai file sottoposti a rotazione sia per il log degli accessi che per il log del server.

  1. Impostare la proprietà di sistema com.sun.enterprise.server.logging.max_history_files sul numero massimo di file di log sottoposti a rotazione da conservare.

    Per informazioni su come impostare una proprietà di sistema, vedere Per gestire le proprietà di sistema.

    Di seguito è indicato il funzionamento della proprietà di sistema com.sun.enterprise.server.logging.max_history_files.

    • Se la proprietà non è impostata, GlassFish Server conserva 10 file di log sottoposti a rotazione al massimo.

    • Se la proprietà è impostata su un numero non valido o su un valore nullo, GlassFish Server conserva 10 file di log sottoposti a rotazione al massimo.

    • Se la proprietà è impostata su 0, GlassFish Server non conserva alcun file di log sottoposto a rotazione.
