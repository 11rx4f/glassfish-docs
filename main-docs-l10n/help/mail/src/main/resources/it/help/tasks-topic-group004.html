Per configurare il runtime batch

  1. Nella struttura di navigazione, selezionare il cluster o l'istanza server di GlassFish Server per cui si desidera visualizzare le esecuzioni di processi batch.

    Viene visualizzata la pagina Informazioni generali.

  2. Selezionare la scheda Batch.

    Viene visualizzata la pagina Esecuzioni processi batch.

  3. Fare clic sulla scheda Configurazione.

    Viene visualizzata la pagina Configurazione runtime batch. Per eseguire i processi batch, il runtime utilizza un servizio esecutore gestito e un'origine dati.

  4. Dall'elenco a discesa Nome ricerca servizio esecutore, selezionare il servizio esecutore gestito da utilizzare per fornire i thread ai processi batch.

    Il servizio esecutore gestito predefinito è concurrent/__defaultManagedExecutorService. Dopo l'invio di un processo batch al runtime batch, è possibile modificare il servizio esecutore gestito senza alcun impatto sull'esecuzione del processo.

  5. Dall'elenco a discesa Nome ricerca origine dati, selezionare l'origine dati da utilizzare per memorizzare le informazioni sui processi.

    L'origine dati predefinita è jdbc/__TimerPool. Non modificare l'origine dati dopo che il primo processo batch è stato inviato al runtime batch per l'esecuzione. Se è necessario modificare l'origine dati, arrestare e riavviare il dominio, quindi apportare la modifica prima dell'avvio o riavvio di qualsiasi processo. Tuttavia, una volta modificata l'origine dati, le informazioni memorizzate nell'origine dati precedente diventano inaccessibili.

  6. Fare clic su Salva.
