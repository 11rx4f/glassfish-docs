Sessioni Jakarta Mail

GlassFish Server include l'interfaccia API Jakarta Mail. Jakarta Mail consente di accedere dalle applicazioni Java ai server di posta che supportano i protocolli IMAP (Internet Message Access Protocol) e SMTP (Simple Mail Transfer Protocol) nella rete locale o su Internet.

Utilizzare la pagina Sessioni Jakarta Mail per configurare le sessioni Jakarta Mail.

Per ogni sessione Jakarta Mail vengono fornite le informazioni riportate di seguito.

Nome JNDI

Il nome univoco che identifica la sessione Jakarta Mail.

Abilitato

Questa colonna viene visualizzata se nel dominio esiste solo l'istanza server predefinita, server.

Un segno di spunta indica che la risorsa è abilitata, una X indica che la risorsa è disabilitata.

Stato

Questa colonna viene visualizzata se nel dominio sono stati creati cluster o istanze standalone.

Il numero di cluster e istanze standalone a cui la risorsa è destinata e in quante di queste destinazioni la risorsa è abilitata. Ad esempio, "Abilitato in 2 di 4 destinazione/i" significa che la risorsa è destinata a quattro cluster e istanze standalone e che è abilitata in due di queste quattro destinazioni.

Descrizione

La descrizione della sessione Jakarta Mail.

La tabella Sessioni contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare una nuova sessione Jakarta Mail.

Elimina

Pulsante che consente di eliminare una o più sessioni Jakarta Mail selezionate.

Abilita

Pulsante che consente di abilitare una o più sessioni Jakarta Mail selezionate.

Disabilita

Pulsante che consente di disabilitare una o più sessioni Jakarta Mail selezionate.
