Editar Sesión Jakarta Mail

Utilice la página Editar sesiones Jakarta Mail para editar una sesión Jakarta Mail.

La página Editar sesiones Jakarta Mail contiene las siguientes opciones.

Cargar Valores por Defecto

Botón para restaurar configuración que tiene valores por defecto a sus valores por defecto. Las configuraciones que no tienen valores por defecto no se modifican.

Nombre JNDI

Nombre único que identifica las sesiones Jakarta Mail. El nombre JNDI es un campo de sólo lectura. Únicamente puede especificar el nombre JNDI al crear una nueva sesión Jakarta Mail.

Host de correo

Host del servidor de correo por defecto. Los métodos de conexión de los objetos de almacenamiento y transporte utilizan este valor si no se suministra una propiedad de host específica del protocolo. El nombre debe poder resolverse en un nombre de host real.

Usuario por defecto

Nombre de usuario que se debe proporcionar cuando se conecta a un servidor de correo. Los métodos de conexión de los objetos de almacenamiento y transporte utilizan este valor si no se suministra una propiedad de nombre de usuario específica del protocolo. El nombre de usuario debe contener únicamente caracteres alfanuméricos, de subrayado, guiones o puntos.

Dirección de remitente por defecto

Dirección de correo electrónico del usuario por defecto. El formato es nombre_usuario@host.dominio.

Orden de despliegue

Orden de despliegue del recurso de sesión de Jakarta Mail.

Descripción

Descripción de esta sesión de correo.

Estado

Indica si el recurso está disponible en el tiempo de ejecución.

Si sólo existe la instancia de servidor por defecto, server en el dominio, la disponibilidad en tiempo de ejecución del recurso puede estar activada o desactivada. La disponibilidad en tiempo de ejecución está activa por defecto.

Si se ha creado algún cluster o alguna instancia independiente en el dominio, utilice el separador Destino para controlar la disponibilidad del recurso en tiempo de ejecución en sus destinos.

Protocolo de almacenamiento

Servicio de protocolo de almacenamiento, que se conecta con un servidor de correo, recupera mensajes y los guarda en una o varias carpetas. Los valores permitidos son imap, pop3, imaps y pop3s. El valor por defecto es imap.

Clase de protocolo de almacenamiento

Clase de implantación del proveedor de servicios para el almacenamiento. Los valores permitidos son:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

El valor por defecto es com.sun.mail.imap.IMAPStore.

Protocolo de transporte

Servicio de protocolo de transporte, que envía mensajes. Los valores permitidos son smtp y smtps. El valor por defecto es smtp.

Clase de protocolo de transporte

Clase de implantación del proveedor de servicios para el transporte. Los valores permitidos son:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

El valor por defecto es com.sun.mail.smtp.SMTPTransport.

Depurar

Si esta opción está seleccionada, la depuración para este recurso está activada, Si el nivel de log de Jakarta Mail se ha definido como FINE o FINER, la salida de depuración se genera y se incluye en el archivo log del sistema. Consulte Para configurar niveles de log para obtener más información sobre la definición del nivel de log.

Propiedades Adicionales

Propiedades adicionales de las sesiones Jakarta Mail. Para obtener una lista de las propiedades disponibles, consulte la documentación de la API de Jakarta Mail (http://java.sun.com/products/mail/javadocs/index.html). Las propiedades adicionales deben comenzar por mail- y como separadores se deben utilizar guiones en lugar de puntos. Por ejemplo, mail-debug es correcta, pero mail.debug no lo es. El backend de GlassFish Server convierte los guiones en los puntos que espera la API de Jakarta Mail.
