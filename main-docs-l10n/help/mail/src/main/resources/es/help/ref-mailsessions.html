Sesiones de Jakarta Mail

GlassFish Server incluye la API de Jakarta Mail. Jakarta Mail proporciona acceso desde aplicaciones Java a servidores de correo compatibles con Internet Message Access Protocol (IMAP) y Simple Mail Transfer Protocol (SMTP) situados en su red o en Internet.

Utilice la página Sesiones Jakarta Mail para configurar sesiones Jakarta Mail.

Para cada sesión Jakarta Mail, se proporciona la siguiente información.

Nombre JNDI

Nombre único que identifica las sesiones Jakarta Mail.

Activado

(Esta columna sólo se muestra si la instancia del servidor por defecto, server, existe en el dominio).

Una marca de control si el recurso está activado, o una X si está desactivado.

Estado

(Esta columna sólo se muestra si en el dominio se ha creado algún cluster o instancia independiente).

Número de clusters e instancias independientes a los que se dirige el recurso y el número de estos destinos en los que el recurso está activado. Por ejemplo, "Activado en 2 de 4 de destinos" significa que el recurso se dirige a cuatro clusters e instancias independientes y que está activado en dos de estos cuatro destinos.

Descripción

Descripción de las sesiones Jakarta Mail.

La tabla Sesiones también contiene las siguientes opciones.

Nuevo

Botón para crear una nueva sesión Jakarta Mail.

Suprimir

Botón para suprimir una o más sesiones Jakarta Mail seleccionadas.

Activar

Botón para activar una o más sesiones Jakarta Mail seleccionadas.

Desactivar

Botón para desactivar una o más sesiones Jakarta Mail seleccionadas.
