Sessões Jakarta Mail

O GlassFish Server inclui a API de Jakarta Mail. Jakarta Mail oferece acesso, a partir de aplicações Java, a servidores de correio capazes de Internet Message Access Protocol (IMAP) e Simple Mail Transfer Protocol (SMTP) em sua rede ou na Internet.

Utilize a página Sessões Jakarta Mail para configurar sessões Jakarta Mail.

Para cada usuário, as informações a seguir são fornecidas.

Nome JNDI

Um nome exclusivo que identifica a sessão Jakarta Mail.

Ativado

(Esta coluna só será exibida se a instância do servidor default, server, existir no domínio.)

Exibe uma marca de seleção se o recurso estiver ativado ou um X se o recurso estiver desativado.

Status

(Esta coluna só será exibida se os clusters ou instâncias standalone tiverem sido criadas no domínio.)

O número de clusters e instâncias independentes ao qual o recurso está direcionado e em quantos desses o recurso está ativado. Por exemplo, "Ativado em 2 de 4 Alvos" significa que o recurso está direcionado para quatro clusters e instâncias standalone e que está ativado em dois desses quatro alvos.

Descrição

Uma descrição da sessão Jakarta Mail.

A tabela Sessões também contém as opções a seguir.

Novo

Botão para criar uma nova sessão Jakarta Mail.

Deletar

Botão para deletar uma ou mais sessões Jakarta Mail selecionadas.

Ativar

Botão para ativar uma ou mais sessões Jakarta Mail selecionadas.

Desativar

Botão para desativar uma ou mais sessões Jakarta Mail selecionadas.
