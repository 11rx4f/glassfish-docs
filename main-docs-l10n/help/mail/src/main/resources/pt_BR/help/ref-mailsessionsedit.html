Editar Sessão Jakarta Mail

Utilize a página Editar Sessão Jakarta Mail para editar uma sessão Jakarta Mail.

A página Editar Sessão Jakarta Mail contém as opções a seguir.

Carregar Defaults

Botão para restaurar definições para seus valores default. Definições que não tiveram seus valores default alterados.

Nome JNDI

Um nome exclusivo que identifica a sessão Jakarta Mail. O nome JNDI é um campo somente para leitura. Você somente pode especificar o nome JNDI quando cria uma nova sessão Jakarta Mail.

Host de Correio

O nome do host do servidor de correio default. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de host específica do protocolo. O nome deve ser resolúvel para um nome de host real.

Usuário Default

O nome de usuário a ser fornecido ao conectar-se a um servidor de correio. Os métodos de conexão dos objetos Armazenamento e Transporte usam esse valor se não é fornecida uma propriedade de nome de usuário específica do protocolo. O nome de usuário deve conter somente caracteres alfanuméricos, sublinhado, traço ou ponto.

Endereço Default do Remetente

O endereço de e-mail do usuário default. O formato é

Nome do usuário@host.domain.

Ordem de Implantação

A ordem de implantação do recurso de sessão do Jakarta Mail.

Descrição

Uma descrição dessa sessão de correio.

Status

Se o recurso está disponível no runtime.

Se apenas a instância do servidor default, server, existir no domínio, a disponibilidade de runtime do recurso poderá ser ativada ou desativada. A disponibilidade de runtime está ativada por default.

Se clusters ou instâncias standalone tiverem sido criados no domínio, utilize a guia Alvo para controlar a disponibilidade de runtime do recurso em seus alvos.

Protocolo de Armazenamento

O serviço de protocolo de armazenamento, que se conecta a um servidor de correio, recuperar mensagens e as salva em uma ou mais pastas. Os valores permitidos são imap, pop3, imaps e pop3s. O valor default é imap.

Classe do Protocolo de Armazenamento

A classe de implementação do provedor do serviço para armazenamento. Os valores permitidos são:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

O valor default é com.sun.mail.imap.IMAPStore.

Protocolo de Transporte

O serviço de protocolo de transporte, que envia mensagens. Os valores permitidos são smtp e smtps. O valor default é smtp.

Classe do Protocolo de Transporte

A classe de implementação do provedor do serviço de transporte. Os valores permitidos são:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

O valor default é com.sun.mail.smtp.SMTPTransport.

Depuração

Se essa opção for selecionada, a depuração para esse recurso é ativada. Se o nível de log Jakarta Mail estiver definido como BOM ou melhor, a saída de depuração será gerada e incluída no arquivo de log do sistema. Para obter informações sobre a definição do nível de log, consulte Para Configurar Níveis de Log.

Propriedades Adicionais

Propriedades adicionais para a sessão Jakarta Mail. Para obter uma lista das propriedades disponíveis, consulte a documentação da API Jakarta Mail (http://java.sun.com/products/mail/javadocs/index.html). Propriedades adicionais devem começar com mail- e devem usar hífens, não pontos, como separadores. Por exemplo, mail-debug é correto, mas mail.debug não. O back-end do GlassFish Server converte os hífens nos pontos esperados pela Jakarta Mail API.
