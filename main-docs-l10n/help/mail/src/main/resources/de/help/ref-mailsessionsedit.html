Jakarta Mail-Session bearbeiten

Auf der Seite "Jakarta Mail-Session bearbeiten" können Sie die Einstellungen einer Jakarta Mail-Session ändern.

Die Seite "Jakarta Mail-Session bearbeiten" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zur Wiederherstellung der Standardwerte von Einstellungen, die Standardwerte haben. Einstellungen, die keine Standardwerte haben, werden nicht geändert.

JNDI-Name

Ein eindeutiger Name, der die Jakarta Mail-Session identifiziert. Der JNDI-Name ist ein schreibgeschütztes Feld. Sie können den JNDI-Namen nur festlegen, wenn Sie eine neue Jakarta Mail-Session erstellen.

Mailhost

Der Hostname des Standard-Mailservers. Die Verbindungsmethoden der Speicher- und Transportobjekte verwenden diesen Wert, wenn keine protokollspezifische Hosteigenschaft angegeben ist. Der Name muss in einen tatsächlichen Hostnamen auflösbar sein.

Standardbenutzer

Der Benutzername, der bei der Verbindungsherstellung zu einem Mailserver angegeben werden muss. Die Verbindungsmethode der Speicher- und Transportobjekte verwendet diesen Wert, wenn keine protokollspezifische Benutzernamenseigenschaft angegeben ist. Der Benutzername darf nur alphanumerische Zeichen, Unterstriche, Gedankenstriche oder Punkte enthalten.

Standardabsenderadresse

Die E-Mail-Adresse des Standardbenutzers. Das Format ist username@host.domain.

Deployment-Reihenfolge

Die Deployment-Reihenfolge der Jakarta Mail-Sessionressource.

Beschreibung

Eine Beschreibung der Mailsession.

Status

Gibt an, ob die Ressource zur Laufzeit verfügbar ist.

Wenn in der Domain nur die Standardserverinstanz server vorhanden ist, kann die Laufzeitverfügbarkeit der Ressource aktiviert oder deaktiviert werden. Laufzeitverfügbarkeit ist standardmäßig aktiviert.

Wurden in der Domain Cluster oder Standalone-Instanzen erstellt, können Sie die Laufzeitverfügbarkeit der Ressource auf ihren Zielen über die Registerkarte "Ziel" steuern.

Speicherprotokoll

Der Speicherprotokollservice, der eine Verbindung zu einem Mailserver herstellt, Nachrichten abruft und Nachrichten in mindestens einem Ordner speichert. Zulässige Werte sind imap, pop3, imapsund pop3s. Der Standardwert ist imap.

Speicherprotokollklasse

Die Serviceprovider-Implementierungsklasse für die Speicherung. Zulässige Werte sind:

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

Der Standardwert ist com.sun.mail.imap.IMAPStore.

Transportprotokoll

Der Transportprotokollservice, der Nachrichten sendet. Zulässige Werte sind smtp und smtps. Der Standardwert ist smtp.

Transportprotokollklasse

Die Serviceprovider-Implementierungsklasse für den Transport. Zulässige Werte sind:

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

Der Standardwert ist com.sun.mail.smtp.SMTPTransport.

Debug

Bei aktivierter Option ist das Debugging für diese Ressource aktiviert. Wenn die Jakarta Mail-Logebene auf FINE (Genau) oder höher festgelegt ist, wird eine Debugging-Ausgabe generiert und in die Systemlogdatei geschrieben. Weitere Informationen zur Festlegung der Logebene finden Sie unter So konfigurieren Sie Logebenen.

Weitere Eigenschaften

Zusätzliche Eigenschaften für die Jakarta Mail-Session. Eine Liste der verfügbaren Eigenschaften finden Sie in der Jakarta Mail-API-Dokumentation (http://java.sun.com/products/mail/javadocs/index.html). Zusätzliche Eigenschaften müssen mit mail- beginnen und Bindestriche anstelle von Punkten als Trennzeichen verwenden. Beispiel: Richtig ist mail-debug, während mail.debug falsch ist. Das GlassFish Server-Backend wandelt die Bindestriche in die von der Jakarta Mail-API erwarteten Punkte um.
