Sessions Jakarta Mail

GlassFish Server inclut l'API Jakarta Mail. Jakarta Mail permet, à partir d'applications Java, d'accéder à des serveurs de messagerie IMAP (Internet Message Access Protocol) et SMTP (Simple Mail Transfer Protocol) via votre réseau ou Internet.

La page Sessions Jakarta Mail permet de configurer les sessions Jakarta Mail.

Pour chaque session Jakarta Mail, les informations suivantes sont fournies :

Nom JNDI

Nom unique identifiant la session Jakarta Mail.

Activé

(Cette colonne est affichée si seule l'instance de serveur par défaut server existe dans le domaine.)

Coche si la ressource est activée ou X si la ressource est désactivée.

Statut

(Cette colonne apparaît si des clusters ou des instances autonomes ont été créés dans le domaine.)

Nombre de clusters et d'instances autonomes sur lesquels la ressource est ciblée et nombre de cibles sur lesquelles la ressource est activée. Par exemple, Activé sur 2 cibles sur 4 signifie que la ressource est ciblée sur quatre clusters et instances autonomes et activée sur deux de ces quatre cibles.

Description

Description de la session Jakarta Mail.

La table Sessions contient également les options ci-après.

Nouveau

Bouton permettant de créer une session Jakarta Mail.

Supprimer

Bouton permettant de supprimer les sessions Jakarta Mail sélectionnées.

Activer

Bouton permettant d'activer les sessions Jakarta Mail sélectionnées.

Désactiver

Bouton permettant de désactiver les sessions Jakarta Mail sélectionnées.
