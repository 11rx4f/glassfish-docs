Per impostare il valore Timeout transazione

Per impostazione predefinita nel server non è previsto alcun timeout per le transazioni. Ciò significa che il server attende il completamento delle transazioni a tempo indeterminato. Se si imposta un valore di timeout per le transazioni e una transazione non viene completata nell'intervallo di tempo configurato, GlassFish Server esegue il rollback della transazione.

  1. Espandere il nodo Configurazione nell'albero di navigazione.

  2. Selezionare l'istanza da configurare:

    • Per configurare un'istanza particolare, selezionare il nodo di configurazione dell'istanza. Ad esempio, per l'istanza predefinita server, selezionare il nodo server-config.

    • Per configurare le impostazioni predefinite per tutte le istanze, selezionare il nodo default-config.

  3. Selezionare il nodo Servizio transazioni.

    Viene visualizzata la pagina Servizio transazioni.

  4. Nel campo Timeout transazione digitare il numero di secondi trascorso il quale si verifica il timeout della transazione.

    Il valore predefinito è 0. Questo valore comporta la disabilitazione del timeout delle transazioni.

  5. Fare clic su Salva.

  6. Riavviare GlassFish Server.
