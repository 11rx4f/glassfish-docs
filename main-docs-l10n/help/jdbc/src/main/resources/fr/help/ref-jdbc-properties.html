Propriétés propres aux pools de connexions JDBC

La plupart des pilotes JDBC permettent d'utiliser des listes de propriétés standard pour spécifier l'utilisateur, le mot de passe et d'autres informations de configuration des ressources. Bien que les propriétés soient facultatives par rapport à GlassFish Server, certaines propriétés peuvent être nécessaires pour la plupart des bases de données. Pour plus d'informations, reportez-vous à JDBC 4.0 Standard Extension API.

La modification des propriétés de pilote JDBC nécessite un redémarrage du serveur.

Les propriétés standard suivantes sont communes à la plupart des fournisseurs JDBC. Pour plus d'informations sur les propriétés disponibles pour votre pilote JDBC, consultez la documentation du fournisseur.

user

Spécifie le nom utilisateur permettant la connexion à la base de données.

password

Spécifie le mot de passe permettant la connexion à la base de données.

databaseName

Spécifie la base de données du pool de connexions.

serverName

Spécifie le serveur de base de données du pool de connexions.

port

Spécifie le port sur lequel le serveur de base de données écoute les demandes.

networkProtocol

Spécifie le protocole de communication.

roleName

Spécifie le nom du rôle SQL initial.

datasourceName

Spécifie un élément XADataSource sous-jacent ou un élément ConnectionPoolDataSource si le groupement de connexions est effectué.

description

Spécifie une description textuelle.

url

Spécifie l'URL du pool de connexions. Bien qu'il ne s'agisse pas d'une propriété standard, elle est couramment utilisée.
