Processus d'écoute HTTP

La page Processus d'écoute HTTP permet de configurer les processus d'écoute HTTP.

Un processus d'écoute HTTP est un type de processus d'écoute réseau. Pour accéder aux fonctionnalités avancées, vous pouvez modifier un processus d'écoute HTTP à l'aide des pages Processus d'écoute réseau.

Lorsque vous installez GlassFish Server pour la première fois, il existe trois processus d'écoute HTTP par défaut portant les noms suivants : admin-listener, http-listener1 et http-listener2. http-listener2, qui est configuré avec SSL, est désactivé par défaut.

Pour chaque processus d'écoute HTTP, les informations suivantes sont fournies.

Nom

Nom du processus d'écoute HTTP.

N° de port

Numéro de port sur lequel le processus d'écoute HTTP écoute.

Adresse

Adresse IP utilisée par le processus d'écoute HTTP.

Activé

Valeur True si le processus d'écoute HTTP est activé ou False s'il ne l'est pas.

La table Processus d'écoute HTTP contient également les options ci-après.

Nouveau

Bouton permettant de créer un processus d'écoute HTTP.

Supprimer

Bouton permettant de supprimer les processus d'écoute HTTP sélectionnés.
