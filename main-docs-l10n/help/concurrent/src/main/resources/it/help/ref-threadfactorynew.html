Nuovo thread factory gestito

Utilizzare la pagina Nuovo thread factory gestito per creare una risorsa thread factory gestito.

La pagina Nuovo thread factory gestito contiene le opzioni riportate di seguito.

Nome JNDI

Il nome JNDI univoco che identifica la risorsa. Per convenzione, il nome inizia con un indicatore del tipo di risorsa e una barra, ad esempio concurrent/, jdbc/ o jms/.

Informazioni contesto

I contesti da propagare ai thread: Classloader, JNDI, Security o WorkArea. La propagazione del contesto può essere abilitata o disabilitata. Se è abilitata, i contesti selezionati vengono propagati. Se è disabilitata, nessuno dei contesti viene propagato, anche se sono selezionati. Il valore predefinito è Abilitato, con tutti i quattro contesti selezionati.

Stato

Lo stato della risorsa. La risorsa può essere abilitata o disabilitata. Lo stato predefinito è Abilitato.

Priorità thread

La priorità da assegnare ai thread creati. Il valore predefinito è 5.

Descrizione

La descrizione di questa risorsa.

Proprietà aggiuntive

Proprietà aggiuntive per la risorsa. GlassFish Server non definisce alcuna proprietà aggiuntiva per questo tipo di risorsa. Inoltre, questo tipo di risorsa non utilizza attualmente alcuna proprietà aggiuntiva.

Destinazioni

Questa opzione viene visualizzata se nel dominio sono stati creati cluster o istanze standalone.

I cluster e le istanze standalone per la risorsa. Per spostare le destinazioni desiderate nella colonna Destinazioni selezionate, utilizzare i pulsanti Aggiungi e Aggiungi tutto. Per spostare le destinazioni non necessarie nella colonna Destinazioni disponibili, utilizzare i pulsanti Rimuovi e Rimuovi tutto.
